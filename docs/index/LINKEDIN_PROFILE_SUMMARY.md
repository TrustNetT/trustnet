## LinkedIn Description (Detailed Version)

> **TrustNet** is a blockchain-based decentralized identity and reputation network I built from scratch starting January 2026.
>
> **The Problem**: Centralized platforms own user identities and data. Reputation doesn't follow users. Trust isn't cryptographic.
>
> **The Solution**: TrustNet creates a decentralized trust infrastructure where:
> - **Users own their identity** (Ed25519 cryptographic keypair, client-side key generation)
> - **Reputation is portable** (blockchain-backed, transferred via IBC to other networks)
> - **Networks prevent spam** (zero reputation = automatic exclusion, no moderation teams needed)
> - **Security is non-negotiable** (TLS 1.3, AES-256-GCM encryption, zero-trust design)
> - **Anyone can run a node** (Alpine Linux VM, minimal resources, 1-liner installation)
> - **Everything is modular** (hot-swappable plugins, 2-5 second dev cycles vs 30 min traditional blockchain)
>
> **Technical Stack**:
> - Blockchain: Cosmos SDK + Tendermint (multi-chain IBC)
> - Frontend: React PWA (Progressive Web App—works everywhere)
> - Infrastructure: Alpine Linux VM, QEMU, Caddy reverse proxy
> - Deployment: Automated installer, hybrid discovery (multicast/DNS/static peer)
>
> **Current Phase**: MVP implementation (hybrid discovery system, web UI finalization, testing framework)
>
> **Impact**: Moving Web3 identity from "nice-to-have" to production-grade security standard.
>
> Open source | GitHub: [github.com/Ingasti/TrustNet](https://github.com/Ingasti/TrustNet)
